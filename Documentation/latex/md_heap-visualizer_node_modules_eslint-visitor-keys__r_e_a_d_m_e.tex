\begin{quote}
type\+: {\ttfamily \{ \mbox{[}type\+: string\mbox{]}\+: string\mbox{[}\mbox{]} $\vert$ undefined \}} \end{quote}


Visitor keys. This keys are frozen.

This is an object. Keys are the type of \href{https://github.com/estree/estree}{\texttt{ E\+S\+Tree}} nodes. Their values are an array of property names which have child nodes.

For example\+:

\DoxyCodeLine{console.log(evk.KEYS.AssignmentExpression) // ‚Üí ["left", "right"]}

\doxysubsection*{evk.\+get\+Keys(node)}

\begin{quote}
type\+: {\ttfamily (node\+: object) =$>$ string\mbox{[}\mbox{]}} \end{quote}


Get the visitor keys of a given A\+ST node.

This is similar to {\ttfamily Object.\+keys(node)} of ES Standard, but some keys are excluded\+: {\ttfamily parent}, {\ttfamily leading\+Comments}, {\ttfamily trailing\+Comments}, and names which start with {\ttfamily \+\_\+}.

This will be used to traverse unknown nodes.

For example\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{const node = \{}
\DoxyCodeLine{    type: "AssignmentExpression",}
\DoxyCodeLine{    left: \{ type: "Identifier", name: "foo" \},}
\DoxyCodeLine{    right: \{ type: "Literal", value: 0 \}}
\DoxyCodeLine{\}}
\DoxyCodeLine{console.log(evk.getKeys(node)) // ‚Üí ["type", "left", "right"]}
\end{DoxyCode}


\doxysubsection*{evk.\+union\+With(additional\+Keys)}

\begin{quote}
type\+: {\ttfamily (additional\+Keys\+: object) =$>$ \{ \mbox{[}type\+: string\mbox{]}\+: string\mbox{[}\mbox{]} $\vert$ undefined \}} \end{quote}


Make the union set with {\ttfamily evk.\+K\+E\+YS} and the given keys.


\begin{DoxyItemize}
\item The order of keys is, {\ttfamily additional\+Keys} is at first, then {\ttfamily evk.\+K\+E\+YS} is concatenated after that.
\item It removes duplicated keys as keeping the first one.
\end{DoxyItemize}

For example\+:

\DoxyCodeLine{console.log(evk.unionWith(\{}
\DoxyCodeLine{    MethodDefinition: ["decorators"]}
\DoxyCodeLine{\})) // ‚Üí \{ ..., MethodDefinition: ["decorators", "key", "value"], ... \}}
\end{DoxyCode}
