Converts back a {\ttfamily yargs} argv object to its original array form.

Probably the unparser word doesn\textquotesingle{}t even exist, but it sounds nice and goes well with \href{https://github.com/yargs/yargs-parser}{\texttt{ yargs-\/parser}}.

The code originally lived in \href{https://github.com/moxystudio}{\texttt{ M\+O\+XY}}\textquotesingle{}s Git\+Hub but was later moved here for discoverability.

\doxysection*{Installation}

{\ttfamily \$ npm install yargs-\/unparser}

\doxysection*{Usage}


\begin{DoxyCode}{0}
\DoxyCodeLine{const parse = require('yargs-\/parser');}
\DoxyCodeLine{const unparse = require('yargs-\/unparser');}
\DoxyCodeLine{}
\DoxyCodeLine{const argv = parse(['-\/-\/no-\/boolean', '-\/-\/number', '4', '-\/-\/string', 'foo'], \{}
\DoxyCodeLine{    boolean: ['boolean'],}
\DoxyCodeLine{    number: ['number'],}
\DoxyCodeLine{    string: ['string'],}
\DoxyCodeLine{\});}
\DoxyCodeLine{// \{ boolean: false, number: 4, string: 'foo', \_: [] \}}
\DoxyCodeLine{}
\DoxyCodeLine{const unparsedArgv = unparse(argv);}
\DoxyCodeLine{// ['-\/-\/no-\/boolean', '-\/-\/number', '4', '-\/-\/string', 'foo'];}
\end{DoxyCode}


The second argument of {\ttfamily unparse} accepts an options object\+:


\begin{DoxyItemize}
\item {\ttfamily alias}\+: The \href{https://github.com/yargs/yargs-parser\#requireyargs-parserargs-opts}{\texttt{ aliases}} so that duplicate options aren\textquotesingle{}t generated
\item {\ttfamily default}\+: The \href{https://github.com/yargs/yargs-parser\#requireyargs-parserargs-opts}{\texttt{ default}} values so that the options with default values are omitted
\item {\ttfamily command}\+: The \href{https://github.com/yargs/yargs/blob/master/docs/advanced.md\#commands}{\texttt{ command}} first argument so that command names and positional arguments are handled correctly
\end{DoxyItemize}

\doxysubsection*{Example with {\ttfamily command} options}


\begin{DoxyCode}{0}
\DoxyCodeLine{const yargs = require('yargs');}
\DoxyCodeLine{const unparse = require('yargs-\/unparser');}
\DoxyCodeLine{}
\DoxyCodeLine{const argv = yargs}
\DoxyCodeLine{    .command('my-\/command <positional>', 'My awesome command', (yargs) =>}
\DoxyCodeLine{        yargs}
\DoxyCodeLine{        .option('boolean', \{ type: 'boolean' \})}
\DoxyCodeLine{        .option('number', \{ type: 'number' \})}
\DoxyCodeLine{        .option('string', \{ type: 'string' \})}
\DoxyCodeLine{    )}
\DoxyCodeLine{    .parse(['my-\/command', 'hello', '-\/-\/no-\/boolean', '-\/-\/number', '4', '-\/-\/string', 'foo']);}
\DoxyCodeLine{// \{ positional: 'hello', boolean: false, number: 4, string: 'foo', \_: ['my-\/command'] \}}
\DoxyCodeLine{}
\DoxyCodeLine{const unparsedArgv = unparse(argv, \{}
\DoxyCodeLine{    command: 'my-\/command <positional>',}
\DoxyCodeLine{\});}
\DoxyCodeLine{// ['my-\/command', 'hello', '-\/-\/no-\/boolean', '-\/-\/number', '4', '-\/-\/string', 'foo'];}
\end{DoxyCode}


\doxysubsection*{Caveats}

The returned array can be parsed again by {\ttfamily yargs-\/parser} using the default configuration. If you used custom configuration that you want {\ttfamily yargs-\/unparser} to be aware, please fill an \href{https://github.com/yargs/yargs-unparser/issues}{\texttt{ issue}}.

If you {\ttfamily coerce} in weird ways, things might not work correctly.
