\href{https://www.typescriptlang.org/}{\texttt{ Type\+Script}} is a language for application-\/scale Java\+Script. Type\+Script adds optional types to Java\+Script that support tools for large-\/scale Java\+Script applications for any browser, for any host, on any OS. Type\+Script compiles to readable, standards-\/based Java\+Script. Try it out at the \href{https://www.typescriptlang.org/play/}{\texttt{ playground}}, and stay up to date via \href{https://blogs.msdn.microsoft.com/typescript}{\texttt{ our blog}} and \href{https://twitter.com/typescript}{\texttt{ Twitter account}}.

Find others who are using Type\+Script at \href{https://www.typescriptlang.org/community/}{\texttt{ our community page}}.

\doxysection*{Building}

In order to build the Type\+Script compiler, ensure that you have \href{https://git-scm.com/downloads}{\texttt{ Git}} and \href{https://nodejs.org/}{\texttt{ Node.\+js}} installed.

Clone a copy of the repo\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{git clone https://github.com/microsoft/TypeScript.git}
\end{DoxyCode}


Change to the Type\+Script directory\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{cd TypeScript}
\end{DoxyCode}


Install \href{https://gulpjs.com/}{\texttt{ Gulp}} tools and dev dependencies\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{npm install -\/g gulp}
\DoxyCodeLine{npm install}
\end{DoxyCode}


Use one of the following to build and test\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{gulp local             \# Build the compiler into built/local.}
\DoxyCodeLine{gulp clean             \# Delete the built compiler.}
\DoxyCodeLine{gulp LKG               \# Replace the last known good with the built one.}
\DoxyCodeLine{                       \# Bootstrapping step to be executed when the built compiler reaches a stable state.}
\DoxyCodeLine{gulp tests             \# Build the test infrastructure using the built compiler.}
\DoxyCodeLine{gulp runtests          \# Run tests using the built compiler and test infrastructure.}
\DoxyCodeLine{                       \# Some low-\/value tests are skipped when not on a CI machine -\/ you can use the}
\DoxyCodeLine{                       \# -\/-\/skipPercent=0 command to override this behavior and run all tests locally.}
\DoxyCodeLine{                       \# You can override the specific suite runner used or specify a test for this command.}
\DoxyCodeLine{                       \# Use -\/-\/tests=<testPath> for a specific test and/or -\/-\/runner=<runnerName> for a specific suite.}
\DoxyCodeLine{                       \# Valid runners include conformance, compiler, fourslash, project, user, and docker}
\DoxyCodeLine{                       \# The user and docker runners are extended test suite runners -\/ the user runner}
\DoxyCodeLine{                       \# works on disk in the tests/cases/user directory, while the docker runner works in containers.}
\DoxyCodeLine{                       \# You'll need to have the docker executable in your system path for the docker runner to work.}
\DoxyCodeLine{gulp runtests-\/parallel \# Like runtests, but split across multiple threads. Uses a number of threads equal to the system}
\DoxyCodeLine{                       \# core count by default. Use -\/-\/workers=<number> to adjust this.}
\DoxyCodeLine{gulp baseline-\/accept   \# This replaces the baseline test results with the results obtained from gulp runtests.}
\DoxyCodeLine{gulp lint              \# Runs eslint on the TypeScript source.}
\DoxyCodeLine{gulp help              \# List the above commands.}
\end{DoxyCode}
